Device Bound Session Credentials (DBSC) for SvelteKit

Bind SvelteKit session cookies to a hardware key so a stolen cookie can't be replayed on another device. dbsc-toolkit ships a first-class SvelteKit adapter, with a Web Crypto polyfill so Firefox and Safari are covered too.

View on GitHub npm All frameworks

DBSC — shipped to Chrome stable in 146 — binds a session to a private key generated inside the device's hardware (a TPM on Windows, the Secure Enclave on Apple Silicon). The key never leaves the device, so a cookie copied to another machine fails the next signature check and the session dies. The dbsc-toolkit/sveltekit adapter wires the whole protocol into SvelteKit.

Install

npm install dbsc-toolkit @sveltejs/kit

Use it in SvelteKit

// src/hooks.server.ts
import { dbscHandle } from "dbsc-toolkit/sveltekit";
import { MemoryStorage } from "dbsc-toolkit/storage/memory";
export const handle = dbscHandle({ storage: new MemoryStorage() });

// in an action / +server:  await bindSession(event, sessionId, storage, { userId });

Every browser, one server

Chromium 145+ binds in hardware (tier: "dbsc"); Firefox, Safari and older Chromium use a non-extractable Web Crypto key in IndexedDB (tier: "bound"). The same requireProof() guard enforces both. Run native-only with bound: false.
